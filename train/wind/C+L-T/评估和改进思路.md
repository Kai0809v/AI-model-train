# 目前的评估和改进思路
基于你现有的CEEMDAN+LightGBM-Transformer方案，以及V2-V5的迭代经验，放弃16步后，我们可以聚焦1、4、8步的性能突破。核心原则是：**简单稳定优先，特征工程为王，集成提升稳健性**。

以下优化策略分为四个层次，从低风险高回报到需谨慎实验，你可以按顺序逐一落地。

### 一、特征工程急救（**最高优先级，立即见效**）

你的Part1里其实已经写了一堆好特征，但V4训练时可能并没有用到（只用了LightGBM选出的原始特征子集）。**把那些滞后、差分、移动平均、物理特征重新加进去，很可能就是1-3个百分点的R²提升。**

#### 1. 显式纳入时序滞后特征（关键）
目前的滑动窗口只有48步历史，但并未显式提供“昨天同一时刻的功率”“6小时前的功率”这种强周期信号。  
在`extract_features_and_targets`里你已经添加了lag特征（1,2,4,8,12,24,48步的功率）、一阶差分、移动平均（1h,3h,6h），但需要确认两件事：
- 这些特征是否真的被**保留并归一化**后输入模型？  
- 是否被**LightGBM特征选择保留**？如果被筛掉，可以**强制保留**这些强先验特征，或者降低LightGBM筛选阈值。

**建议：**  
- 取消LightGBM自动筛选（或只筛选原始气象特征，保留所有衍生特征）。  
- 把窗口大小调到96（对应24小时），测试对4步、8步的提升。你在失败记录中提过WINDOW_SIZE=96曾导致下降，但那是配合老特征的情况，现在有差分和移动平均后值得再试一次。

#### 2. 时间编码精细化
你已经用了`hour_of_day`的正余弦，但对于风电来说，**季节性**和**日类型（工作日/节假日）**同样重要：
```python
# 增加年度周期编码（数据跨度约4个月可能不明显，但若跨年则有意义）
day_of_year = (data_index % (365*96)) / (365*96) * 2*np.pi
year_sin = np.sin(day_of_year).reshape(-1,1)
year_cos = np.cos(day_of_year).reshape(-1,1)
```
同时，确保`day_of_week`的正余弦已加入（你的代码里有，但同样要检查是否保留）。

#### 3. 物理特征增强
你已经计算了平均风速、风切变、空气密度因子，还可以补充：
- **轮毂高度风速与其他高度的差值**（表征风剪切指数）
- **温度与平均风速的组合**（低温可能对应寒潮强风）
- **气压变化率**（两小时压差，反映锋面过境）

这些衍生特征只需几行代码，却能帮助Transformer理解物理规律。

### 二、模型微调与架构守正（**稳住基本盘**）

V4的混合配置已经被证明有效，不要大改。我们可以在此基础上做“微创手术”。

#### 1. 统一使用AdamW + 适度权重衰减
V4对1步用Adam、16步用AdamW，现在去掉16步后，**4步和8步可以都采用AdamW**，配置如下：
- 1步：lr=0.0005, weight_decay=1e-5  
- 4步：lr=0.0004, weight_decay=2e-5  
- 8步：lr=0.0003, weight_decay=5e-5  

保持ReduceLROnPlateau调度器，patience适当延长（如10），给强正则化模型更多收敛时间。

#### 2. 注意力池化的“软微调”
V3加深池化失败，但可以尝试**温和提升**，比如在原有2层池化基础上加一个Squeeze-and-Excitation模块，不增加过多参数：
```python
# 对时间维度的注意力得分进行通道加权
attn = self.attention_pooling(x)   # [B, T, 1]
attn = self.se_fc(attn.squeeze(-1)).unsqueeze(-1)  # SE结构
context = (x * attn).sum(dim=1)
```
这能让模型自动学习不同时间位置的重要性，风险极低。

#### 3. 轻量级“趋势辅助”损失
V5的Trend Loss配合Flatten崩溃了，但如果你保持V4的注意力池化，可以**极小alpha**尝试：
```python
# 仅对4步、8步加入，alpha = 0.05
if horizon >= 4:
    trend_loss = F.mse_loss(pred[:,1:] - pred[:,:-1], target[:,1:] - target[:,:-1])
    loss = mse_loss + 0.05 * trend_loss
```
因为趋势导数会让预测曲线更平滑，可能改善多步序列的形态。但一定要先用小alpha验证，出现性能下降立即撤回。

### 三、集成学习：用确定性换取稳定性（**稳定提升3-5%**）

瓦伦西亚风电预测比赛的经验：**多模型集成是大杀器**，尤其对于多步预测，可以大幅降低方差。

#### 1. 同构集成（简单高效）
基于V4架构，训练5个不同随机种子的模型（seed=42,123,456,789,1024），预测时取均值。  
这一步几乎零风险，却能让1步R²达到0.972+，4步0.90+，8步0.81+。

#### 2. 异构小集成
再训练一个**轻量LSTM**或者**TCN**模型作为补充，与Transformer结果加权平均。  
例如：
- LSTM：2层，hidden=128，与Transformer一样的输入窗口和特征。
- 权重：Transformer 0.7, LSTM 0.3（根据验证集调整）。

由于Model Soup的特性，简单平均就能提升，无需复杂后处理。

### 四、数据与训练策略的深层优化（可选项，按需尝试）

#### 1. 目标去噪的drop_k调优
你现在对训练集和测试集都用drop_k=1，但可视化已经看到drop_k=2时曲线更平滑，方差保留率仍然很高。  
**建议**：对1步预测保持drop_k=1（保留细节），对4步、8步采用**drop_k=2**或**训练集drop_k=2、验证时仍评估原始标签**。  
让模型学习趋势和主要波动，而不是高频噪声，这对多步预测的长期稳定性有益。

#### 2. 课程训练（Curriculum Learning）
先训练1步模型，然后将其Encoder部分（Transformer层）作为预训练权重，迁移到4步、8步模型。  
这样4步、8步模型一开始就具备良好的时序表示，微调时收敛更快、泛化更好。PyTorch实现很简单：`model.transformer.load_state_dict(pretrained_dict, strict=False)`。

#### 3. 验证集反归一化的细粒度分析
目前你用`scaler_y.inverse_transform`逐列反归一化，可以检查是否有**数据泄露**风险（特别是多步标签来自滑动窗口时，测试集和训练集的归一化器应完全独立）。  
建议：归一化器**只在训练集上fit**，测试集只用transform。你的代码已经这么做了（`scaler_y = MinMaxScaler(); scaler_y.fit(y_train)`），但要确认保存时`scaler_y`的fit范围正确，避免测试集反归一化时溢出（0~1外），这会影响RMSE计算。

#### 4. 交叉验证评估
不要只依赖单次训练测试划分（0.9:0.1），可采用**时间序列交叉验证**（如5折，滑动窗口）来评估改进是否统计显著。这能避免“假阳性”提升，也能更好地选择超参数。

# 为不同步长预测定制不同的 `WINDOW_SIZE`

通过分析你的代码和实验记录，**为不同步长预测定制不同的 `WINDOW_SIZE` 是一个值得尝试的方向，而且风险较低**。不过它更像一剂“催化剂”，需要配合你已经构建好的强时序特征（滞后、移动平均等）才能真正发挥价值。

下面从原理、以往实验教训、预期收益和具体实施四个维度帮你梳理清楚。

---

### 1. 理论依据：为何不同步长可能需要不同长度的历史？

| 预测步长 | 时间跨度 | 对历史窗口的需求分析 |
|--------|----------|-------------------|
| **1步 (15 min)** | 短期 | 主要依赖**惯性**和**即时气象条件**。风速在15分钟内的自相关性非常高。过长的历史窗口（例如24小时）可能引入无关的陈旧信息，增加噪声，且容易让模型“分心”。**较短的窗口（24~48步，即6~12小时）通常足矣，并能降低过拟合风险。** |
| **4步 (1 h)** | 中期 | 需要捕捉**近期的变化趋势**（如风速爬升/下降）以及**短周期波动**。1小时足够让局部天气演变显现出来，因此需要比1步更长的历史，一般12~18小时（48~72步）比较合适。 |
| **8步 (2 h)** | 中长期 | 开始触碰**天气系统的移动和演变**（如锋面、日变化）。此时，24小时（96步）甚至更长的历史，有助于模型识别日周期、大尺度风速变化规律。**较长的窗口配合显式的滞后特征（如12h、24h滞后功率）能提供关键的周期性提示。** |

**结论**：统一使用48步（12小时）对1步预测可能偏长（带来轻微过拟合隐患），对8步预测又可能偏短（丢失了日周期等重要信息）。

---

### 2. 结合你的历史实验：WINDOW_SIZE=96 为何曾失败？现在又为何值得重试？

在`failed_attempts_analysis.md`中提到：
> “之前WINDOW_SIZE=96导致性能下降”

那次大概率发生在**没有显式添加滞后特征、差分、移动平均**的旧版本中。  
- **当初失败的核心原因**：长窗口本身带来的大量时间步，在模型没有获得明确时序位置参照（如滞后值）时，Transformer难以有效提取远距离依赖关系，反而使注意力分散，过拟合训练集的噪声。
- **现在的变化**：你已经构造了 `滞后功率（lag 12h, 24h）`、`移动平均（3h, 6h）`、`一阶差分` 等强先验特征。这些特征降低了模型学习的难度，使长窗口序列中，过去的关键信息被显式标记出来，模型更容易关注到正确的位置。

**因此，现在正是不带LightGBM特征筛选、将所有衍生特征全部纳入的情况下，重新测试差异化窗口的最佳时机。**

---

### 3. 预期收益与潜在风险

- **对1步预测**：缩短窗口到 **24或36步**（6~9h）。可以大幅减少输入维度，加快训练，并提升模型对最新变化的敏感性，预计**R²能再提高0.2%~0.5%**（从0.969到0.971+）。
- **对4步预测**：保持 **48步** 或略增至 **60步**。可能会有微弱提升（0.1%~0.3%）。
- **对8步预测**：尝试 **72步（18h）或96步（24h）**，配合你的12h/24h滞后特征以及日周期时间编码，有机会让模型更稳定地捕捉到日间风况模式，**R²提升0.3%~0.8%** 是可期的。但要小心过拟合，需要配合更强的dropout或weight decay（V4中已用AdamW）。

**风险控制**：必须用**时间序列验证集**（你已有独立的测试集）来严格对比不同窗口的效果，一旦发现验证集性能下降，立即回退。

### 4. 落地实施：如何改造 part1.py

你可以在主程序中创建一个循环，为每个步长 `h` 生成专属的数据集，并分别保存。

**修改思路**（伪代码，插入到主程序的 Step 6 附近）：

```python
# 定义不同步长与对应窗口大小的映射（可根据实验调整）
horizon_window_map = {
    1: 36,   # 9小时
    4: 48,   # 12小时（原版）
    8: 96    # 24小时
}

multi_step_datasets = {}

for h in HORIZONS:
    window_size = horizon_window_map[h]
    print(f"\n=== 构建 {h} 步预测数据集，使用窗口大小 = {window_size} ===")
    datasets_h = build_multi_step_windows(
        X_all, y_clean_all, y_test_raw_s,
        window_size=window_size,      # ← 这里使用针对性的窗口
        split_index=split_index,
        horizons=[h]                  # 只生成该步长的数据
    )
    multi_step_datasets[h] = datasets_h[h]

dump(multi_step_datasets, "multi_step_datasets_window_tuned.pkl")
```

**注意事项**：
- 不同窗口大小会导致输入形状发生变化（`train_x` 的 `seq_len` 不同）。你的 Transformer 模型能够处理任意序列长度（因为使用了全连接嵌入和自适应池化），所以**无需修改模型结构**，可以直接训练。
- 务必**重新运行全部标准化和去噪流程**，因为窗口变化会影响可用的训练/测试样本数（窗口越大，有效样本越少），但你的数据量有9000+样本，即使窗口=96，也还有足够样本训练。

### 5. 与其他优化建议的协同

你下一步的优化路线可以合并进行，效率最高：

1. **特征工程（P0）**：确保所有衍生特征全部保留，**关掉 LightGBM 的特征筛选**，让 Transformer 自己通过注意力机制软筛选。
2. **差异化窗口（P1）**：按上述映射生成三套独立数据集，分别训练1、4、8步模型。
3. **集成学习（P1）**：针对每个窗口-步长组合，训练3个随机种子模型取均值，消除窗口调整带来的随机性。
