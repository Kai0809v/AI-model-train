## 📦 GUI 运行所需依赖库清单

根据代码分析，GUI程序需要以下Python库：

### **核心依赖**
1. **PySide6** - Qt GUI框架
2. **pandas** - 数据处理
3. **numpy** - 数值计算（版本 >= 2.0.0）
4. **scipy** - 科学计算（用于图表渲染等，版本 >= 1.13.0）
5. **torch** - PyTorch深度学习框架（版本 2.6.0）
6. **joblib** - 模型序列化/反序列化（版本 >= 1.3.0）
7. **scikit-learn** - 机器学习库（**隐式依赖**，被 joblib 加载模型时间接调用）
8. **openpyxl** - Excel文件读写（用于读取 .xlsx 格式的数据文件）
9. **matplotlib** - 数据可视化库（用于生成预测结果图表、功率曲线等）

### **可选依赖（如果需要使用打包功能）**
10. **PyInstaller** - Python应用打包工具（版本 6.12.0）

---

## ⚠️ 重要说明：为什么需要这些库？

#### **为什么需要 scikit-learn？**
虽然项目代码中没有直接 `import sklearn`，但 **`joblib` 在加载模型文件时会间接依赖它**：

- 项目使用 `joblib.load()` 加载预处理器配置文件（如 `StandardScaler.pkl`、`MinMaxScaler.joblib` 等）
- 这些文件内部保存了 `sklearn.preprocessing` 的对象
- `joblib` 反序列化时会动态导入 `sklearn` 模块来重建这些对象
- **如果不安装 scikit-learn，会在预测时报错：`ModuleNotFoundError: No module named 'sklearn'`**

#### **为什么需要 openpyxl？**
- pandas 读取 `.xlsx` 格式的 Excel 文件时需要 `openpyxl` 引擎
- 项目中用户上传的光伏/风电测试数据通常是 `.xlsx` 格式
- **如果不安装 openpyxl，读取 Excel 文件时会报错：`Missing optional dependency 'openpyxl'`**

#### **为什么需要 matplotlib？**
- 用于生成预测结果的可视化图表（折线图、散点图、残差分布图等）
- 项目中的功率曲线绘制、历史数据分析等功能都依赖 matplotlib
- 虽然 GUI 使用 PySide6 显示图表，但底层绘图仍由 matplotlib 完成
- **如果不安装 matplotlib，生成图表时会报错：`ModuleNotFoundError: No module named 'matplotlib'`**

## 🚀 安装指令

在你的新虚拟环境中执行以下命令：

```bash

# 方法3：分步安装（便于排查问题）
pip install PySide6
pip install pandas numpy scipy
pip install torch
pip install joblib scikit-learn
pip install openpyxl matplotlib
```

## ⚠️ 注意事项

1. **PyTorch版本**：项目中使用的是PyTorch 2.6.0，建议保持一致
2. **NumPy版本**：项目使用的是NumPy 2.x系列（2.2.6），**不要降级到1.x**（会导致兼容性问题）
3. **SciPy版本**：建议使用1.13.0或更高版本，确保与NumPy 2.x兼容
4. **scikit-learn版本**：建议使用1.3.0或更高版本，确保与joblib兼容
5. **CUDA支持**：如果你有NVIDIA GPU，强烈建议安装CUDA版本的PyTorch以获得更好的性能
6. **依赖顺序**：建议先安装 `numpy`、`scipy`、`scikit-learn`，再安装 `torch`，避免版本冲突
7. **matplotlib中文字体**：如果需要显示中文图表，可能需要额外配置中文字体（如 SimHei、Microsoft YaHei）
